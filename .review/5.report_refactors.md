### 🔨 REFACTOR-C2-09 — Migrar `RID.View.init` de polling a event-driven

**🎯 Objetivo**: ARCH-C2-005 — Eliminar el último polling setTimeout del sistema

**💻 Implementación sugerida en** `utils/view-controller.js`:
```javascript
init(callback, options = {}) {
    const source = options.source || 'stop';
    const stateKey = source === 'cead' ? 'STATE_DATA_CEAD' : 'STATE_DATA';
    const eventName = source === 'cead' ? 'dataCeadLoaded' : 'dataManagerLoaded';

    const execute = () => {
        const S = window[stateKey];
        const C = source === 'cead' ? window.COLS_CEAD : window.COLS;
        Promise.resolve(callback(S, C)).catch(err => LOG.error('RID.View.init error:', err));
    };

    if (window[stateKey]?.isLoaded) {
        queueMicrotask(execute); // ya listo
    } else {
        window.addEventListener(eventName, execute, { once: true });
    }
}
```

**⚠️ Riesgo**: Medio — todas las vistas que usan `RID.View.init` deben probarse
**⏱️ Esfuerzo**: L (requiere regression testing en las 25+ vistas)
