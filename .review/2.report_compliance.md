# 🛡️ REPORTE DE CUMPLIMIENTO Y SEGURIDAD: ALGARROBO_BASE

**❗ Hallazgo**: Ausencia de SRI (Subresource Integrity)
**🔓 Vulnerabilidad**: Ataque de Supply Chain
**📍 Ubicación**: `dashboard.html:15, 26, 148-152`
**⚠️ Severidad**: Alta
**📜 Estándar/Ley**: OWASP Top 10 (A06:2021 – Vulnerable and Outdated Components)
**🛠️ Mitigación**: Generar hashes SHA y añadir el atributo `integrity` a todos los scripts cargados desde CDNs (Tailwind, Chart.js, JSpdf, etc.).

**❗ Hallazgo**: Inyección de Código Dinámico (Script Injection)
**🔓 Vulnerabilidad**: DOM-based XSS
**📍 Ubicación**: `js/app.js:372` (método `executeScripts`)
**⚠️ Severidad**: Media/Alta
**📜 Estándar/Ley**: OWASP Top 10 (A03:2021 – Injection)
**🛠️ Mitigación**: Evitar la creación manual de tags `<script>` desde `textContent` de fragmentos cargados vía `fetch`. Se recomienda usar módulos nativos o una whitelist estricta de funciones de inicialización.

**❗ Hallazgo**: Exposición de Secretos (Security by Obscurity)
**🔓 Vulnerabilidad**: Secret Exposure
**📍 Ubicación**: `js/ia2.js:19-21`
**⚠️ Severidad**: Crítica (Si la clave tiene costes o límites)
**📜 Estándar/Ley**: OWASP (A07:2021 – Identification and Authentication Failures)
**🛠️ Mitigación**: Las claves de API nunca deben estar en el frontend. Implementar un Proxy en el Backend que maneje la autenticación con BigModel y que el cliente consuma este endpoint interno.

**❗ Hallazgo**: Almacenamiento de PII en LocalStorage persistente
**🔓 Vulnerabilidad**: Data Leak / Privacy Risk
**📍 Ubicación**: `js/ia2.js:75`
**⚠️ Severidad**: Media
**📜 Estándar/Ley**: GDPR / Ley 19.628 (Chile)
**🛠️ Mitigación**: El análisis de IA contiene nombres de comunas y métricas que, aunque públicas, se asocian a un `codcom` específico. Verificar si los análisis de IA contienen micro-datos que puedan identificar patrones sensibles. Implementar cifrado de datos en `localStorage` si es necesario.
