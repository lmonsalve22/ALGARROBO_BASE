## COMP-C2-001 — XSS via `executeScripts()` en `app.js` (riesgo arquitectónico confirmado)

**❗ Hallazgo**: `App.executeScripts(container)` inyecta dinámicamente etiquetas `<script>` con el contenido de scripts embebidos en vistas cargadas vía `fetch()`. Si el servidor que sirve las vistas está comprometido o la ruta es manipulable, se ejecutaría código arbitrario en el contexto del usuario.

**🔓 Vulnerabilidad**: DOM-based XSS / Arbitrary Code Execution

**📍 Ubicación**: `app.js` → función `executeScripts()`

```javascript
// Patrón peligroso:
scripts.forEach(script => {
    const newScript = document.createElement('script');
    newScript.textContent = script.textContent; // ← contenido sin validar
    document.body.appendChild(newScript);       // ← ejecución inmediata
});
```

**⚠️ Severidad**: Alta

**📜 Estándar**: OWASP A03:2021 – Injection

**🛠️ Mitigación**:
1. **Whitelist de rutas permitidas**: Validar que la URL de `fetch` pertenezca siempre al mismo origen (`window.location.origin`).
2. **CSP `script-src 'self'`**: Aunque la etiqueta creada dinámicamente es `textContent` (no `src`), una CSP estricta con `require-trusted-types-for 'script'` bloquearía la inyección.
3. **Alternativa arquitectónica**: Migrar la lógica de vistas a módulos ES importados en lugar de scripts embebidos en HTML.

## COMP-C2-002 — Credenciales expuestas en `data_cead.js`: `codcom` hardcodeado

**❗ Hallazgo**: `data_cead.js` inicializa `STATE_DATA_CEAD.codcom = 5602` y `targetCod = codcom_url_cead ? parseInt(codcom_url_cead) : 5602`. Esto expone el código de comuna por defecto (ALGARROBO = 5602) directamente en el source, permitiendo que un atacante consulte datafiles de cualquier comuna con solo cambiar el parámetro `?codcom=`.

**🔓 Vulnerabilidad**: Data Exposure / Insecure Configuration

**📍 Ubicación**: `data_cead.js` → líneas 8, 295

**⚠️ Severidad**: Media

**📜 Estándar**: OWASP A05:2021 – Security Misconfiguration

**🛠️ Mitigación**: El `codcom` debe provenir de una sesión autenticada en el backend. Mientras tanto, la exposición es baja si los datafiles de otras comunas no contienen PII; documentar el riesgo aceptado explícitamente.

## COMP-C2-003 — Supply Chain: SRI ausente en CDNs (pendiente de Ciclo 1)

**❗ Hallazgo**: Los atributos `crossorigin="anonymous"` ya fueron añadidos (Ciclo 1), pero los hashes `integrity` aún están ausentes. Sin SRI, un CDN comprometido puede servir JavaScript malicioso que se ejecutaría con plena confianza.

**🔓 Vulnerabilidad**: Supply Chain Attack

**📍 Ubicación**: `dashboard.html` → `<script src="https://cdn.jsdelivr.net/npm/chart.js" defer crossorigin="anonymous">`

**⚠️ Severidad**: Alta

**📜 Estándar**: OWASP A08:2021 – Software and Data Integrity Failures

**🛠️ Mitigación**: Generar hashes SRI actualizados usando `https://www.srihash.org/` o el script local:
```bash
curl -s https://cdn.jsdelivr.net/npm/chart.js/dist/chart.umd.min.js | openssl dgst -sha384 -binary | openssl base64 -A
# Resultado → integrity="sha384-XXXX"
```
Añadir `integrity="sha384-..."` a cada `<script>` CDN en `dashboard.html`.

## COMP-C2-004 — `data_cead.js`: CORS implícito sin validación de origen en fetch

**❗ Hallazgo**: `fetch(fileUrl)` en `dataLoaderCead.load()` no especifica `credentials: 'same-origin'` ni valida que `fileUrl` sea del mismo origen. Si el `codcom_url_cead` es manipulado con un path traversal (`?codcom=../../config`), podría intentar fetches a rutas no previstas.

**🔓 Vulnerabilidad**: Path Traversal / SSRF client-side

**📍 Ubicación**: `data_cead.js` → línea 302

**⚠️ Severidad**: Media

**📜 Estándar**: OWASP A01:2021 – Broken Access Control

**🛠️ Mitigación**:
```javascript
// ANTES
const targetCod = codcom_url_cead ? parseInt(codcom_url_cead) : 5602;

// DESPUÉS — validar que sea un entero positivo de rango esperado
const rawCod = parseInt(codcom_url_cead, 10);
const targetCod = (Number.isInteger(rawCod) && rawCod > 1000 && rawCod < 99999)
    ? rawCod : 5602;
```

## COMP-C2-005 — Datos PII potencialmente cacheados en `STATE_DATA_CEAD` global

**❗ Hallazgo**: `STATE_DATA_CEAD` almacena datos estadísticos comunales (sin PII individual). Sin embargo, `pdf.js` → `generateAISummaryText()` accede a `IAModule.interpretations.vista1`, que puede contener inferencias sobre patrones delictuales específicos de una comuna. Si este texto es registrado en analytics o logs externos, constituye PII indirecta.

**🔓 Vulnerabilidad**: PII / Data Privacy

**📍 Ubicación**: `pdf.js` → `generateAISummaryText()` línea 614

**⚠️ Severidad**: Baja (dado que los datos son agregados, no individuales)

**📜 Estándar**: GDPR Art. 4 / 5

**🛠️ Mitigación**: Documentar explícitamente que las interpretaciones de IA son datos estadísticos agregados y no constituyen PII bajo el contexto de uso.
