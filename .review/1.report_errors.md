## ERR-C2-001 — Memory Leak: `setInterval` no cancelado en `pdf.js`

**❗ Problema**: `PDFModule.timeUpdateInterval` se asigna mediante `setInterval` en `showEnhancedOverlay()`. Si el proceso PDF falla antes de llamar a `hideEnhancedOverlay()`, el interval queda activo indefinidamente, intentando actualizar nodos DOM ya eliminados.

**📍 Ubicación**: `pdf.js` → línea 717 / `hideEnhancedOverlay()` → línea 790

**💥 Escenario de Fallo**:
1. Usuario inicia export PDF.
2. `jsPDF` lanza excepción en el primer `addImage` (ej. imagen corropta).
3. El `catch` de `exportReport` no llama a `hideEnhancedOverlay()`.
4. `setInterval` sigue corriendo, `getElementById('pdfElapsedTime')` retorna `null`, `elapsedEl.textContent` arroja TypeError silencioso.

**⚠️ Impacto**: Alto

**🛠️ Fix Sugerido**:
```javascript
// En hideEnhancedOverlay(), ya correcto. El problema está en exportReport:
} catch (error) {
    this.hideEnhancedOverlay(); // ← Garantizar llamada aquí también
    LOG.error('PDF Error:', error);
} finally {
    this.hideEnhancedOverlay(); // ← Mover a finally para garantía absoluta
}
```

## ERR-C2-002 — Falsy Edge Case: `Math.max(...allPeriodos)` con array vacío en `data_cead.js`

**❗ Problema**: La línea `Math.max(...allPeriodos)` con el operador spread falla silenciosamente cuando `allPeriodos` es vacío, retornando `-Infinity`, que se asigna a `STATE_DATA_CEAD.periodoId`. Las vistas que comparan `periodoId` contra valores reales obtienen siempre `false`.

**📍 Ubicación**: `data_cead.js` → línea 340

**💥 Escenario de Fallo**: CEAD entrega datos donde todas las filas con `delito === 'Total'` tienen `id_periodo === null` (corrupción parcial del JSON).

**⚠️ Impacto**: Medio

**🛠️ Fix Sugerido**:
```javascript
// ANTES
const maxPeriodo = allPeriodos.length > 0 ? Math.max(...allPeriodos) : 0;

// DESPUÉS — guard explícito para evitar -Infinity
const maxPeriodo = allPeriodos.length > 0
    ? allPeriodos.reduce((a, b) => Math.max(a, b), 0)
    : 0;
```

## ERR-C2-004 — `view-controller.js`: polling `setTimeout(100)` sin escape ante `dataManagerLoaded` ya disparado

**❗ Problema**: `RID.View.init` espera que `window.STATE_DATA.isLoaded` sea `true` mediante polling cada 100ms. Si el evento `dataManagerLoaded` se disparó ANTES de que la vista cargara su script (race condition en cadenas de carga de vistas), el polling termina en timeout (15s).

**📍 Ubicación**: `utils/view-controller.js` → línea 43 (`setTimeout(check, 100)`)

**💥 Escenario de Fallo**:
1. `DataManager` carga rápidamente (< 200ms).
2. `App.loadView('vista5')` carga el HTML de la vista.
3. El script embebido en la vista llama a `RID.View.init(callback)`.
4. `STATE_DATA.isLoaded` ya es `true` → el check inmediato debería resolver.
5. En condiciones de CPU alta, el check puede tardarse el primer ciclo y `setTimeout` se encola, pero esto es menor. El problema real es que si el polling empieza DESPUÉS de que `isLoaded` es `true`, falla la primera vez: **NO**, `check()` se llama síncronamente, así que el caso de race condition aquí es en realidad benigno. Sin embargo, el `console.error` en línea 58 no usa `LOG`.

**⚠️ Impacto**: Bajo (riesgo real mínimo, pero el `console.error` es coherncia de logging)

**🛠️ Fix Sugerido**:
```javascript
// Reemplazar en view-controller.js:
} catch (err) {
    LOG.error('❌ RID.View.init error:', err);  // usar LOG en lugar de console
}
// Y la línea de confirmación de carga:
LOG.info('✅ RID.View Controller loaded.');     // usar LOG en lugar de console
```

## ERR-C2-005 — `UIHelper.setHtml()`: innerHTML sin sanitización

**❗ Problema**: `UIHelper.setHtml(id, html)` asigna directamente `innerHTML` con contenido recibido como parámetro. Si alguna vista pasa datos provenientes de `STATE_DATA` (enriquecidos con datos CEAD/STOP sin sanear), se abre un vector XSS.

**📍 Ubicación**: `utils/ui-helper.js` → línea 70

**💥 Escenario de Fallo**: Datos del servidor CEAD con un campo `delito` conteniendo HTML malicioso → alguna vista invoca `UIHelper.setHtml('titulo', row[COLS.DELITO])` → ejecución de scripts.

**⚠️ Impacto**: Medio (el riesgo real depende de si el llamador usa datos de servidor; el fix es preventivo)

**🛠️ Fix Sugerido**:
```javascript
// Añadir variante segura que usa textContent cuando es posible:
setHtmlTrusted(id, html) {
    // Solo usar cuando el HTML es generado internamente (template literals)
    const el = document.getElementById(id);
    if (el) el.innerHTML = html;
},
// Renombrar setHtml() a setHtmlTrusted() y auditar llamadores
```

**❗ Problema**: El archivo define un fallback de `COLS_CEAD` (líneas 31-47) si no existe, luego **siempre** lo redefine en las líneas 51-271, ignorando cualquier configuración previa desde `config/columns.js`. El bloque `if (!window.COLS_CEAD)` es código muerto.

**⚠️ Impacto**: Bajo (deuda técnica de mantenibilidad)

**🛠️ Fix Sugerido**: Eliminar el bloque `if/else` de líneas 31-50, que es código muerto. La definición canónica debe residir en `config/columns.js`.

## ERR-C2-007 — `pdf.js`: acceso a `window.STATE_DATA` sin null-safety en 4+ métodos

**❗ Problema**: Métodos `addHeader`, `generateBackCover`, `generateExecutiveSummary`, `extractKPIData` y `generateAISummaryText` acceden a `window.STATE_DATA` con fallback `|| {}`, pero luego acceden a propiedades específicas sin encadenamiento opcional (ej. `state.comunaName.charAt(0)`). Si `comunaName` está en el objeto pero es `null`, `charAt` arrojará TypeError.

**📍 Ubicación**: `pdf.js` → líneas 99, 242, 408

```javascript
// FRÁGIL:
const comuna = state.comunaName
    ? (state.comunaName.charAt(0).toUpperCase() + state.comunaName.slice(1).toLowerCase())
    : 'Comuna';

// Si state.comunaName === null (no undefined), la condición truthy falla
// pero null?.charAt(0) nunca se evalúa → no hay error. PERO:
// Si state.comunaName === '' (string vacío), el ternario es falsy → OK.
// Riesgo real: state.comunaName = 0 (número) → charAt(0) arrojaría '0'
```

**⚠️ Impacto**: Bajo (edge case poco probable pero posible durante carga parcial)

**🛠️ Fix Sugerido**:
```javascript
const rawComuna = String(state.comunaName || 'Comuna');
const comuna = rawComuna.charAt(0).toUpperCase() + rawComuna.slice(1).toLowerCase();
```
